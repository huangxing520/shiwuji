# 收纳页面删除守卫与物品计数统一

## Context（背景）

收纳页面（[storage_page.dart](file:///c/Users/bron1117/flutter/wupin/lib/screen/storage/storage_page.dart)）当前的删除逻辑是**级联硬删除**：删除房间/柜体/格子时直接连同其下所有子层级与物品一起删除，没有任何前置物品检查。这会导致用户误删仍装有物品的收纳单元，造成数据丢失。

更深层的问题：收纳卡片上显示的“物品数”/占用率/统计来自 `space_items` 表，但该表**在实际运行中永远为空**——没有 UI 创建它（`SpaceItemActions.addItem` 是死代码，无调用方）、seed 数据为空、WebDAV 只能回放曾经导出过的行。用户真正存放进收纳位置的是主物品表 `items`（通过 `add_item_page` / `item_detail_page` 用 `storageLocationTreeProvider` 写入 `cabinetId`/`slotId`）。

因此本次要同时解决两件事：
1. **删除守卫**：删除前检查当前收纳单元及其所有子层级是否含物品，若有则阻止删除并指明具体子单元。
2. **计数统一**：把收纳卡片“物品数”/占用率/顶部统计改为统计主物品 `items`（+ `space_items` 防御性兼容），让显示口径与守卫口径一致，占用率不再恒为 0。

预期结果：删除含物品的收纳单元时被阻止，弹窗指明“卧室 / 床头柜 / 第一层”这类具体路径与物品数；卡片显示的物品数真实反映主物品。

## 关键设计决策

- **物品口径统一为：主物品 `items`（按 `cabinetId`/`slotId`）+ `space_items`（按 `slot_id`）**。两者相加，既覆盖真实数据（主物品），又防御性兼容 `space_items`。守卫与卡片计数用同一口径，保证一致。
- **不新增 DAO 方法**：直接改写现有 3 个 `itemCount` 方法即可让所有显示自动更新（providers 都通过 `dao.itemCount` 取数）。守卫复用这些方法 + 层级遍历定位阻塞子单元。
- **错误提示**：用 `AlertDialog`，指明具体子单元路径 + 物品数 + 迁移引导。
- **无需 build_runner**：`DeletionBlocker` 是普通类，守卫方法是给现有 Notifier 加实例方法，不涉及 codegen。

## 改动清单

### 1. [lib/daos/slot_dao.dart](file:///c/Users/bron1117/flutter/wupin/lib/daos/slot_dao.dart) — 改写 `itemCount`
`slotDao.itemCount(slotId)` 改为统计 `items WHERE slot_id = ?` + `space_items WHERE slot_id = ?`：
```dart
Future<int> itemCount(String slotId) async {
  final result = await customSelect(
    'SELECT (SELECT COUNT(*) FROM items WHERE slot_id = ?) '
    '+ (SELECT COUNT(*) FROM space_items WHERE slot_id = ?) AS total',
    variables: [Variable.withString(slotId), Variable.withString(slotId)],
  ).get();
  return result.first.read<int>('total');
}
```

### 2. [lib/daos/cabinet_dao.dart](file:///c/Users/bron1117/flutter/wupin/lib/daos/cabinet_dao.dart) — 改写 `itemCount`
`cabinetDao.itemCount(cabinetId)` 改为 `items WHERE cabinet_id = ?` + `space_items`（经该柜体下 slots 聚合）：
```dart
Future<int> itemCount(String cabinetId) async {
  final result = await customSelect(
    'SELECT (SELECT COUNT(*) FROM items WHERE cabinet_id = ?) '
    '+ (SELECT COUNT(*) FROM space_items WHERE slot_id IN '
    '(SELECT id FROM slots WHERE cabinet_id = ?)) AS total',
    variables: [Variable.withString(cabinetId), Variable.withString(cabinetId)],
  ).get();
  return result.first.read<int>('total');
}
```

### 3. [lib/daos/room_dao.dart](file:///c/Users/bron1117/flutter/wupin/lib/daos/room_dao.dart) — 改写 `itemCount`
`roomDao.itemCount(roomId)` 改为 `items WHERE cabinet_id IN (该房间柜体)` + `space_items`（经该房间 slots 聚合）：
```dart
Future<int> itemCount(String roomId) async {
  final result = await customSelect(
    'SELECT (SELECT COUNT(*) FROM items WHERE cabinet_id IN '
    '(SELECT id FROM cabinets WHERE room_id = ?)) '
    '+ (SELECT COUNT(*) FROM space_items WHERE slot_id IN '
    '(SELECT s.id FROM slots s INNER JOIN cabinets c ON s.cabinet_id = c.id '
    'WHERE c.room_id = ?)) AS total',
    variables: [Variable.withString(roomId), Variable.withString(roomId)],
  ).get();
  return result.first.read<int>('total');
}
```
> 注：`sumExpectedItems`（容量）不变；占用率 = itemCount / expected，改后变得有意义。

### 4. [lib/providers/storage_providers.dart](file:///c/Users/bron1117/flutter/wupin/lib/providers/storage_providers.dart) — 新增守卫

新增普通类（放在 Space Items 区块之前）：
```dart
class DeletionBlocker {
  final String path; // 显示路径，如 "卧室 / 床头柜 / 第一层"
  final int count;   // 该阻塞子单元中的物品数
  const DeletionBlocker({required this.path, required this.count});
}
```

在 `RoomActions` 加 `checkRoomDeletion(String roomId)`：遍历该房间下柜体→格子，首个含物品的 slot 返回 `DeletionBlocker(path: '房间 / 柜体 / 格子', count)`；若某柜体无 slot 含物品但 `cabinetDao.itemCount(cab.id) > 0`（直接归属柜体的主物品），返回 `DeletionBlocker(path: '房间 / 柜体', count)`；否则 null。

在 `CabinetActions` 加 `checkCabinetDeletion(String cabinetId)`：遍历该柜体下 slot，首个含物品返回 `DeletionBlocker(path: '柜体 / 格子', count)`；否则若 `cabinetDao.itemCount > 0` 返回 `DeletionBlocker(path: '柜体', count)`；否则 null。

在 `SlotActions` 加 `checkSlotDeletion(String slotId)`：`slotDao.itemCount(slotId) > 0` 则返回 `DeletionBlocker(path: slot名, count)`；否则 null。

> 守卫全部复用改写后的 `dao.itemCount`（已含主物品+space_items），`ref.read(itemDaoProvider)` 不再需要，仅需 `roomDao/cabinetDao/slotDao`（已可用）。

### 5. [lib/screen/storage/storage_page.dart](file:///c/Users/bron1117/flutter/wupin/lib/screen/storage/storage_page.dart) — 删除流程改造

- 新增状态 `bool _deleteChecking = false;`，在 `build` 的 Stack 中加一个与 `_batchProcessing` 同构的加载遮罩（`if (_deleteChecking) Positioned.fill(... CircularProgressIndicator)`）。
- `_confirmDeleteRoom` / `_confirmDeleteCabinet` / `_confirmDeleteSlot` 改为 `Future<void>` 异步：先 `setState(_deleteChecking = true)` → 调对应 `checkXxxDeletion` → `finally` 关闭遮罩 → 若返回 `DeletionBlocker` 调 `_showDeleteBlockedDialog` 并 return；否则走原确认对话框→删除。
- 新增 `_showDeleteBlockedDialog(DeletionBlocker b)`：
  ```dart
  AlertDialog(
    title: const Text('无法删除'),
    content: Text('「${b.path}」中存在 ${b.count} 件物品，请先将物品迁移至其他收纳位置后再删除。'),
    actions: [TextButton(onPressed: pop, child: const Text('我知道了'))],
  )
  ```
- 动作面板（`_showRoomActionSheet` 等）里 “删除” 项的 `onTap` 仍 `Navigator.pop(ctx); _confirmDeleteXxx(...)`（不需 await，fire-and-forget 即可）。
- 删除确认对话框文案去掉“将一并删除其下所有…物品”的误导（守卫保证已无物品），改为“确定删除「X」？此操作不可撤销。”。

> 迁移引导：主物品迁移走 `item_detail_page` 的“修改收纳位置”（`itemsProvider.updateLocation`）；`space_items` 走本页批量迁移。错误弹窗只提示，不内嵌跳转。

2. **单元测试**：`flutter test`（确认未破坏 [test/models/item_test.dart](file:///c/Users/bron1117/flutter/wupin/test/models/item_test.dart) 等现有测试）。
3. **手动端到端**（重点）：

   - 长按柜体B→删除：弹“…「柜体B / 格子C」中存在 1 件物品…”。

4. **回归**：顶部统计横幅“物品总数”现在反映主物品数量；库存页（home）物品总数与收纳页“物品总数”口径不同（前者含未定位物品）属预期。
